# Architecture Rules

Always build modular applications.

Separate concerns clearly.

Business logic belongs inside service classes.

Views must remain thin.

Validation belongs inside serializers.

Database logic belongs inside models and managers.

Permissions should be reusable.

Exceptions should be centralized.

Logging should be centralized.

Configuration should be environment-based.

Avoid tight coupling.

Favor composition over inheritance when appropriate.
